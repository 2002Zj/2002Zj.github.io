Comparable和Comparator的比较和使用

前言

问题

在使用Java编程中，我们经常会遇到一些排序相关的问题，java内置的类库中也为我们提供了许多排序的api可以直接使用，如：数组的Arrays.sort()、集合中的Collections.sort()等等，对于String、Integer等类型实现了Comparable接口的类可以直接使用，例如：

List<String> list = new ArrayList<>();

list.add("a");
list.add("b");
list.add("c");
Collections.sort(list);

那我们要对自定义的对象如：Student、Person等进行排序，又该如何解决呢？

解决

其实Java提供了两个接口Comparable和Comparator，来自定义排序规则。

Java 的 Comparable 接口（ java.lang.Comparable）表示一个可比较对象–即可以与其他对象进行比较的对象。例如，数字是可比较的，字符串也是可以使用字母序比较的等等。Java 中的类型的包装类和一些内置类都已经实现了 Java Comparable 接口，支持进行对象间的比较。我们也可以自己实现 Comparable 接口，使自己定义的类具有可比性。

内容

Comparable接口

通过实体类去实现Comparable接口，重写里面的compareTo()方法来自定义排序

Comparable 接口支持泛型，compareTo() 方法将一个参数化类型的对象作为参数并返回一个 int 值。 返回的 int 值 指示调用 compareTo() 方法的对象是否大于、等于或小于参数对象。

  • 返回正值（1 或更大）表示调用 compareTo() 的对象大于参数对象。

  • 返回零，表示两个对象相等。

  • 返回负值（-1 或更小）表示调用 compareTo() 方法的对象小于参数对象。

    public class Person implements Comparable<Person>{
    private String name;
    private Integer age;
    private Double price;
    private short phone;

    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        this.age = age;
    }

    @Override
    public String toString() {
        return "Person{" +
                "name='" + name + '\'' +
                ", age=" + age +
                '}';
    }

    @Override
    public int compareTo(Person o) {
        //按照age升序排序，age为Integer类型，重写了compareTo方法，所以可以调用compareTo
//        return this.age.compareTo(o.age);
//        return o.age.compareTo(this.age);//按照age降序排序
//        return this.age-o.age;//对于整型数据类型(如int,byte,short)，可以使用算数运算符来计算
//        return this.price.compareTo(o.price);//其它数据类型使用其包装类重写的compareTo方法
        
        //先进行age升序排序，年龄相同再根据名称升序排序
        int i = this.age.compareTo(o.age);
        if (i!=0){
            return i;
        }
        return this.name.compareTo(o.name);
    }
}

Comparator接口

// Comparator接口中只定义了compare()方法
public interface Comparator<T> {

    public int compare(T o1, T o2);
}

通过实现Comparator接口重写compare方法，得到一个比较器，排序时传入比较器，就会按照指定规则进行排序； 该方法会返回一个 int 类型的值，用于表示两个对象中哪个更大。

  • 返回正值意味着第一个对象大于第二个对象。
  • 返回负值意味着第一个对象小于第二个对象。
  • 返回 0 表示两个对象相等。

1.自定义比较器comparator，调用Collections.sort()时，传入comparator

public class ComparatorExample {
    public static void main(String[] args) {
        List<Student> list = new ArrayList<>();
        list.add(new Student("1001", "张三", 18));
        list.add(new Student("1002", "李四", 17));
        list.add(new Student("1003", "王五", 21));
        list.add(new Student("1004", "赵六", 19));
        list.add(new Student("1005", "jack", 19));
        // 1.自定义一个比较器
        Comparator<Student> comparator = new Comparator<Student>() {
            //重写compare方法
            @Override
            public int compare(Student o1, Student o2) {
                //根据id降序排序
                return o2.getStudentId().compareTo(o1.getStudentId());
            }
        };
        Collections.sort(list, comparator);
        System.out.println(list);
    }

}

2.也可以通过匿名内部类的方式传入比较器

//通过实现匿名内部类的方式
Collections.sort(list, new Comparator<Student>() {
    @Override
    public int compare(Student o1, Student o2) { 
        //根据id降序排序，如果o1在前，o2在后，那就是降序排序
        return o2.getStudentId().compareTo(o1.getStudentId());
    }
});
System.out.println(list);

3.和stream流进行搭配使用

//
list.stream()
         .sorted(

          //Comparator.comparing(Student::getStudentId)

          Comparator.comparing(Student::getStudentId).reversed()

//多条件排序
 list.stream()

    }

总结

Comparable 和 Comparator 都是用来实现元素排序的

  • Comparable 是通过重写 compareTo 方法实现排序的，而 Comparator 是通过重写 compare 方法实现排序的；
  • Comparable 必须由自定义类内部实现排序方法，而 Comparator 是外部定义并实现排序的。
